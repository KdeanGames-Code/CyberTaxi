CyberTaxi Game Design Document

Authors: Alex “CyberPilot” Dean and Professor Skyforge

Version 1.1, June 24, 2025

Overview

  • Game Title: CyberTaxi
  • Genre: Strategy/Simulation
  • Platform: Web (Progressive Web App), initially built with HTML, CSS, and JavaScript
  • Target Audience: Gamers, Tesla enthusiasts, strategy fans
  • Setting: Current-day urban environments, starting with Austin, TX
  • Core Concept: Players own and manage a fleet of autonomous Tesla RoboTaxis, competing to earn fares, upgrade operations, and expand while navigating real-world challenges.

Game Play

Objective

Maximize in-game earnings and progress through strategic decisions and mission completions.

Mechanics

  • Fleet Management: Players own and oversee CyberTaxis (e.g., CT-001, CT-002, CT-003), monitoring their status and routes.
  • Fare System: The closest CyberTaxi (across all players) gets the ride, with Tesla taking a share to support the service (percentage TBD).
  • Starting Point: Players purchase a garage to build their company, starting with one charging station. Capacity to add more depends on available power, with construction costs for upgrades.
  • Upgrades: Unlocked by progress milestones (e.g., Real Estate Agent, wireless charging, Superchargers), details to be determined.
  • Purchases: CyberTaxis from an in-game Tesla store (e.g., Model Y, future RoboCab), with a used car market for trading.
  • Leasing/Purchasing: Properties leased (30/60 days/1 year) or bought, subleasable to others for a revenue cut.
  • Wear/Maintenance: Increases with use and conditions, affecting performance; high wear requires towing (cost TBD).
  • Environmental Factors: Weather and traffic impact drive times, with protests as occasional challenges offering potential rewards.
  • Demand: Visual cues guide CyberTaxi placement to high-demand areas.
  • Towing: Available for high wear, with a cost and time penalty.
  • Leaderboards: Track company progress, revenue, distance, and overall performance.

User Interface (UI)

  • Top Menu: Fixed, displaying key info and a help icon (Tesla Cybercab colors).
  • Left Panel: Static, lists CyberTaxis with status, includes toggle buttons.
  • Right Panel: Full-width map with real-time CyberTaxi movement, clickable routes (CyberGold), other players’ taxis (Silver Gray).
  • Property Market: Interface for viewing available locations.
  • Wear Display: Visual indicator per vehicle.
  • Weather/Traffic: Overlay indicators, protest alerts.
  • Tesla Store: Mimics Tesla’s website for purchases.

User Experience (UX)

  • Navigation: Smooth, responsive design with a native feel.
  • Interactivity: Clickable elements for routes, charging, towing.
  • Aesthetics: Tesla Cybercab palette, animations, potential sound effects.
  • Tutorial: Initial level for learning basics.

Art and Sound

  • Visuals: Current-day urban maps, CyberGold markers.
  • Sound: Ambient city noise, interactive cues.

Development Notes

  • Current Status: In development, Austin-focused, aiming for a playable version.
  • Next Steps: Test garage mechanics, refine wear/protest systems, design Tesla store UI.

Future Features

  • Alliances: Shared garages, profit splits.
  • Weather Events: Rain, traffic, protests with strategic options.
  • City Expansion: Post-Austin rollout.
  • Leaderboards: Updates with performance badges.
